Scan model details to confirm whether the model meets your requirements and follows your organization’s configuration and governance rules. Fields update dynamically based on the provider. For provider-specific guidance, see on-screen instructions and provider documentation.

-   **Provider**

    Identifies the platform or service hosting the model. For example, bedrock_converse indicates Amazon Bedrock using the Converse API.


-   **Models**

    Lists the specific model versions available for use (e.g., GPT-4, Claude 3.5 Sonnet).


-   **Endpoint Details**

    Shows the full URL used to send requests through the LiteLLM proxy or gateway.


-   **Endpoint**

    Shows the logical endpoint name mapped to the model. Internal routing systems use this mapping.


-   **Model name**

    Shows the canonical name registered in the system. Tools and APIs reference this value.


-   **Observability**

    Shows model usage metrics over a defined period: completion tokens, prompt tokens, successful requests, failed requests, requests over time.


-   **Model details**

    Shows configuration parameters required to connect to and operate the model: credentials, API base, API model, configurations, gateway parameters, model info parameters.
