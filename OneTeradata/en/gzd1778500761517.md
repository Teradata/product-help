Scan model details in the **Overview** tab to confirm whether the model meets your requirements and follows your organization’s configuration and governance rules. Fields update dynamically based on the provider. For provider-specific guidance, see on-screen instructions and provider documentation.

-   **Provider**

    Identifies the platform or service hosting the model. For example, bedrock_converse indicates Amazon Bedrock using the Converse API.


-   **Models**

    Lists the specific model versions available for use (for example, GPT-4, Claude 3.5 Sonnet).


-   **Endpoint**

    Shows the logical endpoint name mapped to the model. Internal routing systems use this mapping.


-   **Model name**

    Shows the model name registered in the system. Tools and APIs reference this value.


-   **Observability**

    Shows model usage metrics over a specified period: number of total requests sent to the model, number of completion tokens, number of prompt tokens, and a chart to help you monitor request trends.


-   **Model details**

    Shows configuration parameters required to connect to and operate the model: credentials, API base, API model, configurations, gateway parameters, model info parameters.
