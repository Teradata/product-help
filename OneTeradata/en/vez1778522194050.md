Create a virtual key to give a user or application secure access to AI models through the gateway.

**Prerequisites:**

-   At least one model is registered in AI Model Hub.


-   You have administrative access.


-   You know whether the key is for a person or an application.


1.  Select **Virtual Keys** tab > **Create Virtual Key**.


1.  In the **Ownership** section, specify who owns the key:

    -   **Owned by**

        Select the owner type:

        **User**: Assign when an individual needs direct access to AI models. For example, a data scientist running experiments.

        **Service account**: Assign when an application or automated system needs access. For example, a production chatbot.


    -   **User ID**

        Enter the email address of the user.


    -   **Tags**

        Add tags to categorize usage. Separate multiple tags with commas. For example, gpt4, premium, production.


1.  In the **Key details** section, configure model access:

    -   **Key name**

        Enter a descriptive name to identify the key. For example, Production System Key.


    -   **Models**

        Select the models the key can access. To allow access to all models, select All.


1.  In the **Expiration** section, set when the key expires and select the time.


1.  (Optional) To configure rate limits, turn off **Use defaults** and update advanced options:

    -   **Tokens per minute (TPM)**

        Limits the number of tokens processed per minute. Both input and output tokens count toward this limit. Requests are throttled or rejected when the limit is exceeded.


    -   **Requests per minute (RPM)**

        Limits the number of API requests per minute. Each request counts as one, regardless of size. Requests are rejected when the limit is exceeded.


1.  Select **Create virtual key**.

    **Important**: The key is shown only once. Copy it immediately and share it securely with the intended user. If the key is lost, you must create a new one.
