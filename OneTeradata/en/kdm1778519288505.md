Monitor AI model usage and performance across your organization. Review consumption patterns, track how models and virtual keys are used over time, and identify optimization opportunities.

-   **Total requests**: Cumulative number of requests sent to models.

    When to use it: Measure overall usage growth and estimate cost per request.


-   **Completion tokens**: Total tokens generated by models as responses.

    When to use it: Understand output verbosity and response size.


-   **Prompt tokens**: Total tokens sent as input prompts to models.

    When to use it: Analyze input complexity and prompt engineering efficiency.


-   **Successful requests**: Requests processed successfully with valid responses.

    When to use it: Assess system reliability and model availability.


-   **Failed requests**: Requests that returned errors such as authentication issues, rate limits, or timeouts.

    When to use it: Identify stability, configuration, or integration issues.


-   **Models**: Number of models available in AI Model Hub.

    When to use it: Review gateway capacity and identify unused models.


-   **Keys**: Number of active virtual keys.

    When to use it: Review access distribution and manage key lifecycles.


-   **Top models used**: Models with the highest request volume or token consumption.

    When to use it: Identify the most critical or heavily used models.


-   **Top consuming keys**: Virtual keys ranked by usage or spend.

    When to use it: Review high-consumption users or services.


-   **Requests over time**: Request trends across a selected time range.

    When to use it: Identify spikes, drops, or recurring patterns and plan capacity.
