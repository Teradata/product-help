While creating a vector collection, you can select a language model from the available AWS and Azure models.

If you select AWS, you can choose between Anthropic Claude 3 Haiku and Anthropic Claude 3.5 Sonnet.

If you select Azure, you can use GPT-4.1-mini for Language (Chat) model.

Select Anthropic Claude 3 Haiku for high-volume, low-latency tasks where speed and cost-efficiency are crucial, such as rapid data analysis, instant customer service chatbots, content moderation, generating quick code suggestions, and processing large document sets for quick categorization and metadata extraction, especially when you need fast, reliable, but not necessarily the most complex, reasoning. It excels at tasks requiring rapid response times and handling large datasets quickly, making it ideal for user-facing applications and back-end data processing.

Select Anthropic Claude 3.5 Sonnet for a powerful blend of speed, cost-effectiveness, and strong performance in coding, data analysis, and complex workflows, especially when you need rapid iteration, handling large contexts, or visual understanding without the expense of top-tier models like Opus. It's ideal for developers, data scientists, and businesses needing scalable AI for customer support, code migration, and content creation with human-like nuance.

Select GPT-4.1 Mini, if you require a highly cost efficient and low latency AI model as a smaller alternative to GPT-4 class systems. It is ideal for real-time, high-volume applications and supports an exceptionally large **1 million token context window**, enabling single pass analysis of long documents or large codebases. With multimodal input support and strong reasoning and coding performance, GPT 4.1 Mini is well suited for agents, developer tools, and production scale, cost sensitive environments.
