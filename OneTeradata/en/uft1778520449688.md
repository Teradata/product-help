Register a new Amazon Bedrock model to make it available through the gateway. After you add the model, you can assign it to users through virtual keys.

Before you begin, make sure that:

-   You have administrative privileges.


-   The provider endpoint is added to the allowlist of your environment. If not, see the documentation link on this page to request Teradata to add the endpoint.


1.  Make sure that you have administrative access.


1.  In the **Overview** or **Models** tab, select **Add Model**.


1.  On the **Add model** page, enter the model details:

    -   **Provider**

        Select **AWS Bedrock** to register a model hosted on Amazon Bedrock.


    -   **Gateway model name(s)**

        Enter the model name used when making gateway.completion() or gateway.embedding() calls. This is the name the gateway sends to the provider API. Enter a unique name that does not conflict with existing registered models.


    -   **Mode**

        Select a mode when you add a model to define how it processes requests and which API endpoint it uses.

        These modes are available:

        |Mode|What it does|Usage Examples|
        |-----|-------------|---------------|
        |Chat - /chat/completions|Conversational interactions with context retention across requests.|Support assistants, virtual agents, Q&A systems.|
        |Completion - /completions|Single-prompt text generation. No conversation history.|Content generation, autocompletion, document drafting.|
        |Embedding - /embeddings|Generates vector embeddings from text. Returns vectors, not text.|Semantic search, similarity detection, classification.|


1.  In the **Credentials** section, provide the required authentication and connection information:

    -   **AWS access key ID**

        The access key ID from your AWS IAM credentials. Required.


    -   **AWS secret access key**

        The secret access key paired with your AWS access key ID. Required.


    -   **AWS session token**

        A temporary session token for AWS STS authentication. Required only when using temporary credentials.


    -   **AWS region name**

        The AWS region where the Bedrock model is deployed (for example, us-west-2). Ensure the region matches the model's deployment location. Required.


    -   **AWS session name**

        A session name for tracking and auditing when assuming an IAM role. Optional.


    -   **AWS profile name**

        The named profile from your AWS credentials file. Use this when multiple AWS profiles are configured. Optional.


    -   **AWS role name**

        The IAM role ARN to assume for cross-account or restricted access to Bedrock resources. Optional.


    -   **AWS web identity token**

        The web identity token for OIDC-based identity federation. Required only when using OIDC authentication.


    -   **AWS Bedrock runtime endpoint**

        A custom Bedrock runtime endpoint URL. Use this to override the default regional endpoint if your organization uses a private or custom endpoint. Optional.


1.  Select **Test connection** to verify the API base URL, API key, and model identifier.

    The page shows a success or error result.


1.  (Optional) To edit advanced options, turn off **Use defaults** and update advanced configuration settings. See [Advanced Model Configuration Options](https://www.teradata.com).

    -   **Gateway parameters**

        Define parameters such as rate limits (rpm) and timeouts (timeout) in JSON format.


    -   **Supported parameters**

        Add model-specific metadata for discovery, auditing, or integrations.


1.  Select **Add**.

    The model becomes available for usage and governance in AI Model Hub.
