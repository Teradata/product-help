Erstellen Sie einen virtuellen Schlüssel, um einem Benutzer oder einer Anwendung sicheren Zugriff auf KI-Modelle über das Gateway zu gewähren.

**Voraussetzungen:**

-   Mindestens ein Modell ist im KI-Modell-Hub registriert.


-   Sie verfügen über Administratorzugriff.


-   Sie wissen, ob der Schlüssel für eine Person oder eine Anwendung vorgesehen ist.


1.  Wählen Sie die Registerkarte **Virtuelle Schlüssel** > **Virtuellen Schlüssel erstellen** aus.


1.  Geben Sie im Abschnitt **Eigentum** an, wer Eigentümer des Schlüssels ist:

    -   **Eigentümer ist**

        Wählen Sie den Eigentümertyp aus:

        **Benutzer**: Weisen Sie dies einem Benutzer zu, der direkten Zugriff auf KI-Modelle benötigt. Beispiel: Ein Datenwissenschaftler, der Experimente durchführt.

        **Dienstkonto**: Weisen Sie dies zu, wenn eine Anwendung oder ein automatisiertes System Zugriff erfordert. Beispiel: Ein Produktions-Chatbot.


    -   **Benutzerkennung**

        Geben Sie die E-Mail-Adresse des Benutzers ein.


    -   **Tags**

        Fügen Sie Tags hinzu, um die Nutzung zu charakterisieren. Trennen Sie mehrere Tags mit Kommas. Beispiel: gpt4, Premium, Produktion.


1.  Konfigurieren Sie im Abschnitt **Schlüsseldetails** den Modellzugriff:

    -   **Schlüsselname**

        Geben Sie einen beschreibenden Name zur Identifizierung dieses Schlüssels ein. Beispiel: Produktions-Systemschlüssel.


    -   **Modelle**

        Wählen Sie die Modelle aus, auf die der Schlüssel Zugriff ermöglicht. Wählen Sie „Alle“ aus, um Zugriff auf alle Modelle zu ermöglichen.


1.  Legen Sie im Abschnitt **Ablauf** fest, wann der Schlüssel abläuft, und wählen Sie die Zeit aus.


1.  (Optional) Wenn Sie Ratenlimits konfigurieren möchten, müssen Sie **Standardeinstellungen verwenden** deaktivieren und die erweiterten Konfigurationseinstellungen aktualisieren:

    -   **Token pro Minute (TPM)**

        Begrenzt die Anzahl an pro Minute verarbeiteten Token. Dies berücksichtigt sowohl Eingabe- als auch Ausgabetoken. Anfragen werden verlangsamt oder abgelehnt, wenn das Limit überschritten wird.


    -   **Anfragen pro Minute (RPM)**

        Begrenzt die Anzahl der API-Anfragen pro Minute. Jede Anfrage zählt als eine, ungeachtet ihrer Größe. Anfragen werden abgelehnt, wenn das Limit überschritten wird.


1.  Wählen Sie **Virtuellen Schlüssel erstellen** aus.

    **Wichtig**: Der Schlüssel wird nur einmal angezeigt. Kopieren Sie ihn sofort und geben Sie ihn auf sichere Weise an den vorgesehenen Benutzer weiter. Bei Verlust des Schlüssels müssen Sie einen neuen erstellen.
