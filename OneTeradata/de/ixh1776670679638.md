Mithilfe der Registerkarte „Observability“ können Sie Metriken und Eigentümerdetails für eine Bereitstellung anzeigen.

Wählen Sie die Bereitstellung aus, um verschiedene Registerkarten anzuzeigen.

**Dashboard**: Die Zusammenfassungskarten zeigen Folgendes an:

- Ablaufverfolgungen insgesamt (verarbeitete Anfragen)

- Tokenanzahl (insgesamt verbrauchte Token)

- P50-Latenz (mediane Reaktionszeit)

- Fehlerrate

**Ablaufverfolgungen**: Anzeigen verfügbarer Versionen, Status usw. Sie können Filter anwenden, um die Ablaufverfolgungssuche zu verfeinern. Sie können den Agenten ausführen und die Eingabe/Ausgabe sowie zugehörige Metadaten anzeigen.

Die folgenden Details werden auf der Registerkarte „Ablaufverfolgungen“ angezeigt:

- Dauer pro Anfrage.

- Eingabe-/Ausgabetoken

- Fehlerdetails für fehlgeschlagene Anfragen

- Spannen zeigen interne Agentenschritte an

**Spanne**: Gibt einen einzelnen Vorgang in einer Ablaufverfolgung an.

**Metriken**: Gibt quantitative Messungen an, die zur Bewertung der Modellleistung verwendet werden.
