Registrieren Sie ein neues Amazon Bedrock-Modell, damit dieses über das Gateway verfügbar wird. Sobald das Modell hinzugefügt ist, können Sie es Benutzern mittels virtueller Schlüssel zuweisen.

Bevor Sie anfangen, müssen Sie sich vergewissern, dass:

-   Sie über Administratorberechtigungen verfügen.


-   Der Anbieterendpunkt zur Positivliste Ihrer Umgebung hinzugefügt wurde. Falls nicht, können Sie Teradata über den Dokumentationslink auf dieser Seite bitten, den Endpunkt hinzuzufügen.


1.  Stellen Sie sicher, dass Sie über Administratorzugriff verfügen.


1.  Wählen Sie auf der Registerkarte **Übersicht** oder **Modelle** die Option **Modell hinzufügen** aus.


1.  Geben Sie auf der Seite **Modell hinzufügen** die Modelldetails ein:

    -   **Anbieter**

        Wählen Sie **AWS Bedrock** aus, um ein auf Amazon Bedrock gehostetes Modell zu registrieren.


    -   **Gateway-Modell-Name(n)**

        Geben Sie den Modellnamen an, der für den Aufruf von gateway.completion() bzw. gateway.embedding() verwendet wird. Dies ist der Name, den das Gateway an die Anbieter-API sendet. Geben Sie einen eindeutigen Namen an, der keinen Konflikt mit vorhandenen registrierten Modellen darstellt.


    -   **Modus**

        Wählen Sie beim Hinzufügen eines Modells einen Modus aus, um zu definieren, wie es Anfragen verarbeitet und welchen API-Endpunkt es verwendet.

        Die folgenden Modi sind verfügbar:

        |Modus|Funktion|Anwendungsbeispiele|
        |-----|-------------|---------------|
        |Chat - /chat/completions|Gesprächsinteraktionen mit Kontextspeicherung über verschiedene Anfragen hinweg.|Support-Assistenten, virtuelle Agenten, Q&A-Systeme.|
        |Vervollständigung - /completions|Texterstellung anhand einzelner Prompts. Kein Konversationsverlauf.|Inhaltserstellung, automatische Vervollständigung, Erstellung von Dokumentenentwürfen.|
        |Einbettung - /embeddings|Erzeugt Vektoreinbettungen aus Text. Gibt Vektoren statt Text zurück.|Semantische Suche, Ähnlichkeitserkennung, Klassifizierung.|


1.  Geben Sie im Abschnitt **Anmeldedaten** die erforderliche Authentifizierungs- und Verbindungsinformationen an:

    -   **AWS-Zugriffsschlüssel-ID**

        Die Zugriffsschlüssel-ID aus Ihren AWS IAM-Anmeldeinformationen. Erforderlich.


    -   **Geheimer AWS-Zugriffsschlüssel**

        Der geheime Zugriffsschlüssel, der mit Ihrer AWS-Zugriffsschlüssel-ID gepaart ist. Erforderlich.


    -   **AWS-Sitzungstoken**

        Ein temporäres Sitzungstoken für die AWS STS-Authentifizierung. Nur erforderlich, wenn temporäre Anmeldeinformationen verwendet werden.


    -   **Name der AWS-Region**

        Die AWS-Region, in der das Bedrock-Modell bereitgestellt wird (z. B. us-west-2). Stellen Sie sicher, dass die Region dem Bereitstellungsort des Modells entspricht. Erforderlich.


    -   **Name der AWS-Sitzung**

        Ein Sitzungsname für Verfolgung und Auditing, wenn eine IAM-Rolle übernommen wird. Optional.


    -   **Name des AWS-Profils**

        Das benannte Profil aus Ihrer Datei der AWS-Anmeldeinformationen. Verwenden Sie dies, wenn mehrere AWS-Profile konfiguriert sind. Optional.


    -   **Name der AWS-Rolle**

        Der ARN der IAM-Rolle, die für kontoübergreifenden oder eingeschränkten Zugriff auf Bedrock-Ressourcen eingenommen wird. Optional.


    -   **AWS-Web-Identitäts-Token**

        Das Web-Identitäts-Token für OIDC-basierten Identitätsverbund. Nur erforderlich, wenn OIDC-Authentifizierung verwendet wird.


    -   **AWS Bedrock-Laufzeitendpunkt**

        Die URL des Endpunkts einer benutzerdefinierten Bedrock-Laufzeit. Verwenden Sie dies, um den standardmäßigen regionalen Endpunkt zu überschreiben, wenn Ihre Organisation einen privaten oder benutzerdefinierten Endpunkt verwendet. Optional.


1.  Wählen Sie **Verbindung testen** aus, um die API-Basis-URL, den API-Schlüssel und den Modell-Bezeichner zu verifizieren.

    Auf dieser Seite wird angezeigt, ob der Vorgang erfolgreich war oder fehlgeschlagen ist.


1.  (Optional) Wenn Sie die erweiterten Optionen bearbeiten möchten, deaktivieren Sie **Standardeinstellungen verwenden** und aktualisieren Sie die erweiterten Konfigurationseinstellungen. Siehe [Erweiterte Modellkonfigurationsoptionen](https://www.teradata.com).

    -   **Gateway-Parameter**

        Definieren Sie Parameter wie Ratenlimits (rpm) und Timeouts (timeout) im JSON-Format.


    -   **Unterstützte Parameter**

        Fügen Sie modellspezifische Metadaten für Ermittlung, Auditing oder Integrationen hinzu.


1.  Wählen Sie **Hinzufügen** aus.

    Das Modell wird zur Nutzung und Steuerung im KI-Modell-Hub verfügbar.
