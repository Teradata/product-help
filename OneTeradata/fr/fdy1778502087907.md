Enregistrez un nouveau modèle d'IA dans la plate-forme de modèles d'IA afin qu'il puisse être utilisé, contrôlé et supervisé par votre organisation.

Avant de commencer, assurez-vous des éléments suivants :

-   Vous disposez de privilèges d'administrateur.


-   Le point de terminaison du fournisseur est ajouté à la liste d'autorisations de votre environnement. Dans le cas contraire, reportez-vous au lien de la documentation sur cette page pour demander à Teradata d'ajouter le point de terminaison.


1.  Dans l'onglet **Présentation** ou **Modèles**, sélectionnez **Ajouter un modèle**.


1.  Sur la page **Ajouter un modèle**, entrez les détails du modèle :

    -   **Fournisseur**

        Sélectionnez le fournisseur d'IA (par exemple, OpenAI ou Anthropic).


    -   **Nom(s) de modèle(s) de la passerelle**

        Entrez un nom interne court et explicite. Utilisez un nom qui reflète l'objet du modèle (par exemple, content-analysis-GPT-5.1).


    -   **Mode**

        Lorsque vous ajoutez un modèle, sélectionnez un mode afin de définir comment il traite les demandes et quel point de terminaison d'API il utilise.

        Les modes suivants sont disponibles :

        |Mode|Ce qu'il fait|Exemples d'utilisation|
        |-----|-------------|---------------|
        |Discussion - /chat/completions|Interactions conversationnelles avec conservation du contexte entre les demandes.|Assistants de soutien, agents virtuels, systèmes Q&R.|
        |Exécution - /completions|Génération de texte à partir d'une invite unique. Pas d'historique de conversation.|Génération de contenu, auto-exécution, ébauche de document.|
        |Intégration - /embeddings|Génère des intégrations vectorielles à partir d'un texte. Renvoie des vecteurs, pas du texte.|Recherche sémantique, détection de similarités, classification.|


1.  Dans la section **Identifiants**, fournissez les informations d'authentification et de connexion requises :

    -   **Base de l'API**

        Saisissez l'URL racine de l'API du fournisseur (par exemple, https://api.openai.com/v1).


    -   **Modèle d'API**

        Saisissez l'identifiant du modèle attendu par le fournisseur (par exemple, gpt-4-turbo). Pour les modèles personnalisés, spécifiez la référence interne ou le chemin du point de terminaison.


    -   **Clé API**

        Saisissez le jeton d'authentification généré par votre fournisseur. Traitez-le comme un identifiant sensible.


1.  Sélectionnez **Tester la connexion** pour vérifier l'URL de base de l'API, la clé API et l'identifiant du modèle.

    La page affiche un résultat de réussite ou d'erreur.


1.  (Facultatif) Pour modifier les options avancées, désactivez l'option **Utiliser les valeurs par défaut** et mettez à jour les paramètres de configuration avancés. Reportez-vous à la section [Options de configuration avancées du modèle](https://www.teradata.com).

    -   **Paramètres de la passerelle**

        Définissez les paramètres facultatifs, tels que les limites de débit (rpm) et les délais d'expiration (timeout) dans le format JSON.


    -   **Paramètres pris en charge**

        Ajoutez les métadonnées spécifiques du modèle pour la découverte, l'audit ou les intégrations.


1.  Sélectionnez **Ajouter**.

    Le modèle peut alors être utilisé et contrôlé dans la plate-forme de modèles d'IA.
