Analysez les détails du modèle dans l'onglet **Présentation** pour vérifier si le modèle répond à vos besoins et respecte les règles de configuration et de gouvernance de votre organisation. Les champs sont mis à jour de façon dynamique en fonction du fournisseur. Pour les indications spécifiques du fournisseur, reportez-vous aux instructions à l'écran et à la documentation du fournisseur.

-   **Fournisseur**

    Identifie la plate-forme ou le service qui héberge le modèle. Par exemple, bedrock_converse indique Amazon Bedrock qui utilise l'API Converse.


-   **Modèles**

    Affiche la liste des versions de modèles spécifiques disponibles (par exemple, GPT-4, Claude 3.5 Sonnet).


-   **Point de terminaison**

    Affiche le nom du point de terminaison logique mappé au modèle. Les systèmes de routage internes utilisent ce mappage.


-   **Nom du modèle**

    Affiche le nom du modèle enregistré dans le système. Les outils et les API référencent cette valeur.


-   **Observabilité**

    Affiche les mesures d'utilisation du modèle sur une période spécifiée : nombre total de demandes envoyées au modèle, nombre de jetons d'exécution, nombre de jetons d'invite et un graphique pour vous aider à surveiller les tendances en matière de demandes.


-   **Détails du modèle**

    Affiche les paramètres de configuration requis pour se connecter au modèle et l'utiliser : identifiants, base d'API, modèle d'API, configurations, paramètres de passerelle, paramètre d'informations sur le modèle.
