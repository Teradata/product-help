Vous pouvez sélectionner les algorithmes de recherche et également définir les paramètres de recherche à partir de cette page.

En fonction de vos sélections, différentes options s'offrent à vous.

## FLAT


Lorsque vous sélectionnez FLAT, dans les paramètres de recherche, vous pouvez conserver les valeurs par défaut ou sélectionner les valeurs requises en désactivant l'option Utiliser les valeurs par défaut. Vous pouvez modifier les éléments suivants :

**Correspondances de similarité** : sélectionnez le nombre (entre 1 et 1 024) de correspondances de similarité à générer. Utilisez une valeur peu élevée si vous avez besoin des correspondances les plus exactes et également pour optimiser les performances et réduire les coûts.

**Seuil de similarité** : permet de prendre en compte les tables et vues correspondantes. Il s'agit d'un filtre de qualité, qui renvoie uniquement les résultats pertinents au-dessus d'un score de similarité spécifique ou au-dessous d'une valeur de correspondance éloignée, au lieu des k meilleures correspondances.

**Paramètres généraux** : sélectionnez Euclidien, Cosinus ou Produit scalaire.

• **Euclidien** : calcule la similarité en utilisant la distance euclidienne entre les vecteurs.

• **Cosinus** : calcule la similarité cosinus (produit scalaire normalisé) entre deux vecteurs. Utilisez ce paramètre pour la recherche sémantique avec intégration de texte, la comparaison de documents et les systèmes de recommandation pour lesquels l'ampleur n'est pas pertinente.

• **Produit scalaire** : calcule la similarité entre vecteurs à l'aide du produit scalaire. Utilisez ce paramètre lorsque la popularité (ampleur) doit avoir la priorité dans le classement ou pour la recherche de vecteurs normalisés.

## IVF_FLAT


Lorsque vous sélectionnez IVF_FLAT, vous pouvez conserver les valeurs par défaut ou sélectionner les valeurs requises en désactivant l'option **Utiliser les valeurs par défaut**. Vous pouvez modifier les éléments suivants :

**Paramètres d'indexation** : sélectionnez les valeurs requises, telles que le nombre de clusters à entraîner, le nombre d'itérations à exécuter, la valeur initiale, l'algorithme d'initialisation des centroïdes (aléatoire ou K-moyennes++), etc. Cette option est utilisée principalement pour accélérer les recherches de similarité en partitionnant les données en clusters, créant ainsi une recherche approximative des voisins les plus proches au lieu d'une analyse exacte et lente.

**Paramètres de recherche** : sélectionnez le cluster d'entraînement à compter, les correspondances de similarité et les valeurs de seuil.

**Paramètres généraux** : sélectionnez Euclidien, Cosinus ou Produit scalaire.

## HNSW


Lorsque vous sélectionnez HNSW, vous pouvez conserver les valeurs par défaut ou sélectionner les valeurs requises en désactivant l'option **Utiliser les valeurs par défaut**. Vous pouvez modifier les éléments suivants :

**Paramètres d'indexation** : sélectionnez le nombre de couches du graphique, le nombre de connexions de nœuds, les valeurs de construction EF, etc.

**Paramètres de recherche** : sélectionnez les correspondances de similarité et les valeurs de recherche EF à utiliser.

**Paramètres généraux** : sélectionnez Euclidien, Cosinus ou Produit scalaire.
