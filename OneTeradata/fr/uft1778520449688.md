Enregistrez un nouveau modèle Amazon Bedrock pour le rendre disponible par le biais de la passerelle. Après avoir ajouté le modèle, vous pouvez l'attribuer à des utilisateurs par le biais de clés virtuelles.

Avant de commencer, assurez-vous des éléments suivants :

-   Vous disposez de privilèges d'administrateur.


-   Le point de terminaison du fournisseur est ajouté à la liste d'autorisations de votre environnement. Dans le cas contraire, reportez-vous au lien de la documentation sur cette page pour demander à Teradata d'ajouter le point de terminaison.


1.  Assurez-vous de disposer de droits d'accès d'administration.


1.  Dans l'onglet **Présentation** ou **Modèles**, sélectionnez **Ajouter un modèle**.


1.  Sur la page **Ajouter un modèle**, entrez les détails du modèle :

    -   **Fournisseur**

        Sélectionnez **AWS Bedrock** pour enregistrer le modèle hébergé sur Amazon Bedrock.


    -   **Nom(s) de modèle(s) de la passerelle**

        Entrez le nom du modèle utilisé lors des appels gateway.completion() ou gateway.embedding(). Il s'agit du nom que la passerelle envoie à l'API du fournisseur. Entrez un nom unique qui n'entre pas en conflit avec les modèles enregistrés existants.


    -   **Mode**

        Lorsque vous ajoutez un modèle, sélectionnez un mode afin de définir comment il traite les demandes et quel point de terminaison d'API il utilise.

        Les modes suivants sont disponibles :

        |Mode|Ce qu'il fait|Exemples d'utilisation|
        |-----|-------------|---------------|
        |Discussion - /chat/completions|Interactions conversationnelles avec conservation du contexte entre les demandes.|Assistants de soutien, agents virtuels, systèmes Q&R.|
        |Exécution - /completions|Génération de texte à partir d'une invite unique. Pas d'historique de conversation.|Génération de contenu, auto-exécution, ébauche de document.|
        |Intégration - /embeddings|Génère des intégrations vectorielles à partir d'un texte. Renvoie des vecteurs, pas du texte.|Recherche sémantique, détection de similarités, classification.|


1.  Dans la section **Identifiants**, fournissez les informations d'authentification et de connexion requises :

    -   **ID de clé d'accès AWS**

        L'ID de clé d'accès de vos identifiants IAM AWS. Obligatoire.


    -   **Clé d'accès secrète AWS**

        Clé d'accès secrète associée à votre ID de clé d'accès AWS. Obligatoire.


    -   **Jeton de session AWS**

        Jeton de session temporaire pour l'authentification STS AWS. Nécessaire uniquement lorsque des identifiants temporaires sont utilisés.


    -   **Nom de la région AWS**

        Région AWS où le modèle Bedrock est déployé (par exemple, us-west-2). Garantit que la région correspond au lieu de déploiement du modèle. Obligatoire.


    -   **Nom de session AWS**

        Nom de session pour le suivi et l'audit lorsqu'il s'agit d'un rôle IAM. Facultatif.


    -   **Nom du profil AWS**

        Profil désigné dans votre fichier d'identifiants AWS. Utilisez cette option lorsque plusieurs profils AWS sont configurés. Facultatif.


    -   **Nom du rôle AWS**

        ARN de rôle IAM pour l'accès entre plusieurs comptes ou restreint aux ressources Bedrock. Facultatif.


    -   **Jeton d'identité Web AWS**

        Jeton d'identité Web pour la fédération d'identité basée sur l'authentification OIDC. Obligatoire uniquement lorsque l'authentification OIDC est utilisée.


    -   **Point de terminaison d'exécution AWS Bedrock**

        URL de point de terminaison d'exécution Bedrock personnalisée. Utilisez cette option pour remplacer le point de terminaison régional par défaut si votre organisation utilise un point de terminaison privé ou personnalisé. Facultatif.


1.  Sélectionnez **Tester la connexion** pour vérifier l'URL de base de l'API, la clé API et l'identifiant du modèle.

    La page affiche un résultat de réussite ou d'erreur.


1.  (Facultatif) Pour modifier les options avancées, désactivez l'option **Utiliser les valeurs par défaut** et mettez à jour les paramètres de configuration avancés. Reportez-vous à la section [Options de configuration avancées du modèle](https://www.teradata.com).

    -   **Paramètres de la passerelle**

        Définissez les paramètres tels que les limites de débit (rpm) et les délais d'expiration (timeout) dans le format JSON.


    -   **Paramètres pris en charge**

        Ajoutez les métadonnées spécifiques du modèle pour la découverte, l'audit ou les intégrations.


1.  Sélectionnez **Ajouter**.

    Le modèle peut alors être utilisé et contrôlé dans la plate-forme de modèles d'IA.
