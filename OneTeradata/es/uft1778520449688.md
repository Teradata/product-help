Registre un nuevo modelo de Amazon Bedrock para ponerlo a disposición a través de la puerta de enlace. Después de agregar el modelo, puede asignarlo a los usuarios a través de claves virtuales.

Antes de comenzar, asegúrese de lo siguiente:

-   Tiene privilegios de administrador.


-   El endpoint del proveedor se agrega a la lista de permitidos de su entorno. En caso contrario, consulte el enlace a la documentación en esta página para solicitar que Teradata agregue el endpoint.


1.  Asegúrese de tener acceso administrativo.


1.  En la pestaña **Descripción general** o **Modelos**, seleccione **Agregar modelo**.


1.  En la página **Agregar modelo**, introduzca los detalles del modelo:

    -   **Proveedor**

        Seleccione **AWS Bedrock** para registrar un modelo alojado en Amazon Bedrock.


    -   **Nombre(s) de modelo de puerta de enlace**

        Introduzca el nombre del modelo utilizado para realizar las llamadas gateway.completion() o gateway.embedding(). Este es el nombre que la puerta de enlace envía a la API del proveedor. Introduzca un nombre único que no entre en conflicto con los modelos registrados existentes.


    -   **Modo**

        Seleccione un modo cuando agregue un modelo para definir cómo las solicitudes y qué endpoint de API usa.

        Están disponibles estos modos:

        |Modo|Qué hace|Ejemplos de uso|
        |-----|-------------|---------------|
        |Conversación: /chat/completions|Interacciones conversacionales con conservación de contexto entre solicitudes.|Asistentes de apoyo, agentes virtuales y sistemas de preguntas y respuestas.|
        |Finalización: /completions|Generación de texto con una sola solicitud. Sin historial de conversaciones.|Generación de contenido, autocompletado y redacción de documentos.|
        |Incrustación: /embeddings|Genera incrustaciones de vectores desde texto. Devuelve vectores, no texto.|Búsqueda semántica, detección de similitudes, clasificación.|


1.  En la sección **Credenciales**, proporcione la información de conexión y autenticación requerida:

    -   **ID de clave de acceso de AWS**

        El ID de clave de acceso de sus credenciales de IAM de AWS. Obligatorio.


    -   **Clave de acceso secreta de AWS**

        La clave de acceso secreta emparejada con su ID de clave de acceso de AWS. Obligatorio.


    -   **Token de sesión de AWS**

        Un token de sesión temporal para autenticación de STS de AWS. Obligatorio solo cuando se usan credenciales transitorias.


    -   **Nombre de región de AWS**

        La región de AWS donde se despliega el modelo de Bedrock (por ejemplo, us-west-2). Asegúrese de que la región coincida con la ubicación de despliegue del modelo. Obligatorio.


    -   **Nombre de sesión de AWS**

        Un nombre de sesión para realizar el seguimiento y la auditoría cuando se asume un rol de IAM. Opcional.


    -   **Nombre de perfil de AWS**

        El perfil con nombre de su archivo de credenciales de AWS. Úselo cuando se configuren varios perfiles de AWS. Opcional.


    -   **Nombre de rol de AWS**

        El ARN del role de IAM para asumir el acceso restringido o entre cuentas a recursos de Bedrock. Opcional.


    -   **Token de identidad web de AWS**

        El token de identidad web para federación de identidad basado en OIDC. Solo es obligatorio cuando se usa la autenticación de OIDC.


    -   **Endpoint de tiempo de ejecución de Bedrock de AWS**

        Una URL de tiempo de ejecución de Bedrock personalizada. Úsela para anular el endpoint regional predeterminado si su organización usa un endpoint personalizado o privado. Opcional.


1.  Seleccione **Probar conexión** para comprobar la URL base de la API, la clave de API y el identificador del modelo.

    La página muestra un resultado erróneo o correcto.


1.  (Opcional) Para editar las opciones avanzadas, desactive **Usar los valores predeterminados** y actualice los ajustes de configuración avanzada. Consulte [Opciones de configuración avanzada de modelos](https://www.teradata.com).

    -   **Parámetros de puerta de enlace**

        Defina los parámetros como límites de velocidad (rpm) y tiempos límite en formato JSON.


    -   **Parámetros admitidos**

        Agregue metadatos específicos del modelo para detección, auditoría e integraciones.


1.  Seleccione **Agregar**.

    El modelo pasa a estar disponible para su uso y gobernanza en el Centro de modelos de IA.
