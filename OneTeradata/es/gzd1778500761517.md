Analice los detalles del modelo en la pestaña **Descripción general** para confirmar si este cumple con sus requisitos y sigue las reglas de gobernanza y configuración de su organización. Los campos se actualizan de forma dinámica según el proveedor. Para pautas específicas del proveedor, consulte las instrucciones en pantalla y la documentación de este.

-   **Proveedor**

    Identifica la plataforma o el servicio que aloja el modelo. Por ejemplo, bedrock_converse indica que Amazon Bedrock usa la API de conversación.


-   **Modelos**

    Enumera las versiones del modelo específicas que están disponibles para su uso (por ejemplo, GPT-4, Claude 3.5 Sonnet).


-   **Endpoint**

    Muestra el nombre del endpoint lógico asignado al modelo. Los sistemas de enrutamiento interno usan esta asignación.


-   **Nombre de modelo**

    Muestra el nombre del modelo registrado en el sistema. Las herramientas y las API toman este valor como referencia.


-   **Observabilidad**

    Muestra las métricas de uso del modelo en un plazo dado: cantidad de solicitudes totales enviadas al modelo, cantidad de tokens de finalización, cantidad de tokens de solicitud y un gráfico para ayudarle a supervisar las tendencias de las solicitudes.


-   **Detalles de modelo**

    Muestra los parámetros de configuración requeridos para conectarse al modelo y operarlo: credenciales, base de API, modelo de API, configuraciones, parámetros de puerta de enlace, parámetros de información del modelo.
