From Elastic Compute, select **Create**.

-   Select the site for the Elastic Compute instance.


-   Create a name for the Elastic Compute instance.


-   Use auto-scale and auto-suspend to help manage your consumption.


-   Auto-scale automatically scales resources as needed within the range you select.


-   Auto-suspend stops compute resources when inactive.


**Show advanced options**

If your region permits, you can select from a pre-provisioned pool of running instances to reduct provisioning time.

To access block storage Elastic Compute you must enable QueryGrid/Teradata Fabric. Enable QueryGrid/Teradata Fabric to run queries across multiple data sources without moving or copying the data.

Viewpoint allows you to monitor up to 10 elastic compute instances.

You can set a schedule to have elastic compute run during certain hours daily or weekly.
