Following are the SAML limitations with Global Idenity.

VantageCloud Enterprise 3.3 currently does not support SAML with Global Identity. Global Identity requires OpenID Connect (OIDC). Support for SAML-based authentication is available in VantageCloud Enterprise up to version 3.2 only, but support is not available in the Global Identity Console flow. There are no SAML configuration pages for new or existing users in the Global Identity setup wizard or IdP management pages.

If your organization currently uses a SAML identity provider, you must register a new OIDC application in your IdP and complete the OIDC configuration in Console before you can use Global Identity. Your existing SAML setup remains active during this transition.

**What Works with SAML**

If you use a SAML identity provider, you can authenticate with your credentials through your existing IdP. SAML fully supports the following capabilities:

-   Console access


-   Mapping roles across all five platform roles


-   SCIM-based user provisioning


-   All control plane service operations


If you use or plan to use Compute Engine, you must configure an OIDC identity provider. Starting with VantageCloud Enterprise 3.3 upgrade, Compute Engine requires JWT tokens that you obtain through Identity Brokering, which relies on an OIDC-compliant identity provider.

**At a glance**

|Capability|SAML 2.0|OIDC|
|-----------|---------|-----|
|Console access and mapping roles|Yes|Yes|
|SCIM user provisioning|Yes|Yes|
|Control plane services|Yes|Yes|
|Compute Engine|No|Yes|
|Identity Brokering|No|Yes|

**Switching from SAML to OIDC - What to Expect**

SAML and OIDC are fundamentally different protocols, so you cannot migrate or convert an existing SAML configuration. To enable OIDC:

1.  Register a new application in your corporate IdP (Microsoft Entra, Okta, Ping, and others.).


1.  Obtain a Client ID and Client Secret for the new application.


1.  Complete the OIDC configuration workflow in the VantageCloud Enterprise Console.


The existing SAML configuration and user access can remain active while you set up OIDC. After the OIDC connection is validated and live, you can retire SAML at your convenience.

As Compute Engine requires OIDC, Teradata recommends beginning this setup before upgrading to the VantageCloud Enterprise 3.3 if Compute Engine is part of your planned adoption.

For transition guidance, contact your Teradata Customer Success team.
