Une [connexion personnelle par défaut](wkm1725389190945.md) est nécessaire pour créer des modèles d'ensembles de données. Teradata recommande de créer une connexion personnelle d'ensemble de données par défaut avant de créer et d'entraîner un modèle. Reportez-vous à [Ajouter une connexion d'ensemble de données](vpe1725389258480.md).

1.  Dans le tableau de bord ModelOps, sélectionnez **Ensembles de données** > **Créer un modèle d'ensemble de données**.


1.  Si vous y êtes invité, sélectionnez la connexion personnelle d'ensemble de données par défaut dans la liste déroulante, puis sélectionnez **Continuer**.


1.  Définissez les propriétés de l'onglet **Catalogue**, puis sélectionnez **Suivant**.

    -   Fournissez un **Nom** et une **Description** significatifs.


    -   Ajoutez des **Balises** pour mieux identifier ce modèle d'ensemble de données.


    -   Sélectionnez un **Catalogue de caractéristiques** pour définir les types de catalogues.

        Le catalogue de caractéristiques répertorie ces dernières dans un magasin.

        Si vous sélectionnez **Aucun**, sélectionnez **Suivant**, renseignez les champs de l'onglet **Configuration**, puis sélectionnez **Créer**.

        -   Dans l'onglet **Paramètres courants**, définissez les **Propriétés personnalisées** : spécifiez les propriétés personnalisées sous la forme d'une paire clé/valeur. Ces propriétés représentent les valeurs par défaut utilisées pour l'entraînement et l'évaluation des étapes du cycle de vie pour tous les niveaux d'étendue.

            Dans **Modèle de requête**, spécifiez un modèle de requête qui extrait les données pour le modèle d'ensemble de données.

            Dans l'onglet **Paramètres de l'étendue**, spécifiez les propriétés pour différents niveaux d'étendue : **Entraîner**, **Évaluer**, **Noter**. Ici, vous pouvez sélectionner un niveau d'étendue et modifier les valeurs par défaut définies dans l'onglet **Paramètres communs**.

            Définissez les **Propriétés personnalisées** : spécifiez les propriétés personnalisées sous la forme d'une paire clé/valeur pour chaque étendue sélectionnée.

            Dans **Modèle de requête**, spécifiez un modèle de requête qui extrait les données pour démarrer les tâches d'entraînement, d'évaluation et de notation.


    -   Si vous sélectionnez **Vantage** pour le catalogue des caractéristiques :

        Dans l'onglet **Catalogue**, développez la zone **Base de données des statistiques de caractéristiques**, spécifiez une **Base de données** et une **Table**.

        Si aucune table n'est créée pour stocker les statistiques de caractéristiques, saisissez un nom de table et sélectionnez **Créer une table de métadonnées**.


1.  Renseignez les champs de l'onglet **Caractéristiques**, puis sélectionnez **Suivant**.

    -   Définissez une **Requête** et sélectionnez **Exécuter** pour l'exécuter sur les caractéristiques extraites directement de la base de données. La table Caractéristiques affiche les résultats.


    -   ModelOps identifie la clé d'entité de la table par une petite icône de clé ![icône de clé d'entité.](Images/pvr1547509678690.png). Utilisez la colonne Caractéristique pour spécifier si une ligne est une caractéristique (activée) ou une clé d'entité (désactivée). Lorsque vous changez de clé d'entité, une confirmation vous est demandée.


    -   Récupérez les résultats de la requête SQL que vous avez définie précédemment en sélectionnant **Prévisualiser les données**.


    -   **Validez les statistiques** pour la sélection actuelle des caractéristiques. ModelOps affiche un avertissement pour les statistiques manquantes des caractéristiques et la possibilité de générer ou de regénérer des statistiques pour les caractéristiques sélectionnées.


1.  Renseignez les champs de l'onglet **Entité et cible**, puis sélectionnez **Suivant**. Sélectionnez la clé d'entité et la cible à l'aide de cet onglet.

    -   Définissez une **Requête** et sélectionnez **Exécuter** pour récupérer les variables. Les résultats obtenus s'affichent dans la table des variables ; cochez ou décochez les cases en regard de chaque ligne pour affiner votre échantillon. La sélection de la clé d'entité et les opérations associées sont identiques à celles de l'étape de l'onglet **Caractéristiques**.


    -   Récupérez un échantillon des résultats des requêtes SQL définies précédemment avec celles obtenues en sélectionnant **Prévisualiser les données**.


    -   **Validez les statistiques** pour la sélection actuelle de la cible. ModelOps affiche un avertissement pour les statistiques manquantes des cibles et la possibilité de générer ou de regénérer des statistiques pour la cible sélectionnée.


1.  Renseignez les champs de l'onglet **Prédictions**.

    Vous inclurez ici la base de données et la table pour stocker les prédictions et la requête. ModelOps utilise cette requête comme entrée de l'exécution de votre modèle en Production lorsque ce modèle sera déployé en tant que LOT.

    Vous pouvez **prévisualiser les données** pour récupérer les résultats de l'échantillon de la requête SQL avec les clés d'entité pour la notation.


1.  Sélectionnez **Créer**.
