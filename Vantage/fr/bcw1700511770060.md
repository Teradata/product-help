Un administrateur d'organisation peut provisionner le service de copie de données sur VantageCloud Lake à l'aide de l'assistance Teradata.

Créez une demande d'assistance sur [https://support.teradata.com](https://support.teradata.com) pour obtenir de l'aide concernant les exigences suivante :

-   Configurez **QueryGrid** pour connecter la copie de données entre les environnements.


-   Installez le gestionnaire de copie sur VantageCloud Lake.


-   Créez une instance AWS ou Azure [point de terminaison d'interface](dvp1707442265467.md) entre les environnements.


-   Installez l'agent de copie sur l'environnement cible. Il peut s'agir d'un environnement VantageCloud Lake, VantageCloud Enterprise ou VantageCore.

    **QueryGrid** est la méthode de copie prise en charge.


-   Connectez les environnements.


**Remarques** : 

-   Vous ne pouvez pas connecter le gestionnaire de copie à un système déjà connecté à un **Gestionnaire de copie de données** différent.


-   Vous ne pouvez pas connecter le gestionnaire de copie à plusieurs systèmes.


Pour obtenir des instructions détaillées sur le provisionnement pour les plates-formes non prises en charge par Lake, reportez-vous à [Activer le service de copie de données](https://docs.teradata.com/access/sources/dita/topic?dita:topicPath=zmv1694773546514.dita&utm_source=console&utm_medium=iph).

## 1. Conditions préalables de QueryGrid


Vous devez effectuer la configuration de **QueryGrid suivante avant d'activer la copie de données :**

1.  Provisionnez **QueryGriddans votre environnement VantageCloud Lake.**

    Dans l'environnement, accédez à **Gérer les données** > **QueryGrid**. Reportez-vous à [Déploiement de QueryGrid](wrw1640282047881.md).


1.  Créez une connexion à la liaison privée pour que **QueryGrid** connecte VantageCloud Lake à un environnement Vantage cible ou à un autre environnement VantageCloud Lake.


1.  Établissez un serveur étranger (cible) pour effectuer des copies de données.

    Un serveur étranger peut être un environnement VantageCloud Lake, VantageCloud Enterprise ou VantageCore


## 2. Ajouter un environnement


Lors du premier provisionnement de la copie de données, sélectionnez un environnement VantageCloud Lake disponible dans le menu.

Lors de l'ajout d'un environnement supplémentaire, accédez à **Copie de données** > **Connexions** > **Carte des connexions** et sélectionnez **+** pour rechercher une source de données établie dans QueryGrid à partir du menu.

Pour connecter un environnement, préparez vos informations sur votre point de terminaison (AWS ou Azure) de la liaison privée.

## 3. Installer le gestionnaire de copie


Le gestionnaire de copie est le module logiciel installé sur VantageCloud Lake pour copier les données vers un environnement cible et à partir de celui-ci.

Sélectionnez l'un des environnements VantageCloud Lake disponibles dans le **Carte des connexions** qui n'affiche pas le gestionnaire de copie comme étant installé. Dans le volet **Détails des environnements**, sélectionnez **Installer le gestionnaire de copie** dans le menu.

## 4. Installer l'agent de copie


L'agent de copie est le module logiciel installé sur les environnements cibles VantageCloud Lake, VantageCloud Enterprise ou VantageCore pour copier des données vers l'environnement source VantageCloud Lake (ou à partir de celui-ci) sur lequel le gestionnaire de copie est installé.

Lorsque vous connectez un environnement, vous installez l'agent de copie dans le cadre de la configuration de la connexion cible.

**Remarque :** L'assistance de Teradata est requise pour installer le logiciel de l'agent de copie de données sur un système VantageCloud Enterprise.
