Select a node pool when you run a job to allocate compute resources that match your workload requirements. Each node pool is a group of compute nodes with defined hardware such as vCPUs, memory, and GPUs.

Available types depend on your environment configuration. For node pool types, see [Resource Settings](jur1778559719371.md).

**Note:** If a node pool includes GPUs, the GPU allocation appears as an additional resource option during configuration.

Select **View Pool Details** to review nodes in the pool and their current resource allocation. The following node states are available:

-   **Available**

    The node is idle and not running any jobs.


-   **In use**

    The node is running and processing one or more jobs.


-   **At capacity**

    The node is fully utilized. No additional resources can be allocated from this node.


Nodes start and stop dynamically based on demand. If all nodes in a pool are at capacity, the job is added to a queue and runs when resources become available.

A portion of each node's resources is reserved for infrastructure operations. The maximum you can allocate for a job is the total node capacity minus the reserved amount. For example, if a node has 64 vCPU and 13 vCPU are reserved, the maximum you can allocate is 51 vCPU.
