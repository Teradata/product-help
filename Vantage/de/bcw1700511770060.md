Ein Organisationsadministrator kann den Datenkopierdienst auf VantageCloud Lake mit Unterstützung des Teradata-Supports bereitstellen.

Erstellen Sie eine Supportanfrage bei [https://support.teradata.com](https://support.teradata.com), um Unterstützung bei den folgenden Anforderungen zu erhalten:

-   Konfigurieren Sie **QueryGrid**, um eine Datenkopieverbindung zwischen Umgebungen herzustellen.


-   Installieren Sie den Copy Manager auf VantageCloud Lake.


-   Erstellen Sie eine AWS- oder Azure-[Schnittstellenendpunkt](dvp1707442265467.md) zwischen Umgebungen.


-   Installieren Sie den Copy Agent in der Zielumgebung. Dies kann entweder eine VantageCloud Lake-, VantageCloud Enterprise- oder VantageCore-Umgebung sein.

    **QueryGrid** ist die unterstützte Kopiermethode.


-   Verbinden Sie die Umgebungen.


**Hinweise**:

-   Sie können Copy Managers mit keinem System verbinden, das bereits mit einem anderen **Copy Manager für Daten** verbunden ist.


-   Der Copy Manager kann nur mit einem System verbunden werden.


Ausführliche Anweisungen zur Bereitstellung für nicht von Lake unterstützte Plattformen finden Sie unter [Data Copy-Dienst aktivieren](https://docs.teradata.com/access/sources/dita/topic?dita:topicPath=zmv1694773546514.dita&utm_source=console&utm_medium=iph).

## 1. Voraussetzungen von QueryGrid


Sie müssen die folgende **QueryGrid-Konfiguration durchführen, bevor die Datenkopie aktiviert werden kann:**

1.  Stellen Sie **QueryGridin Ihrer VantageCloud Lake-Umgebung bereit.**

    Gehen Sie in der Umgebung zu **Daten verwalten** > **QueryGrid**. Weitere Informationen finden Sie unter [QueryGrid-Bereitstellung](wrw1640282047881.md).


1.  Erstellen Sie eine PrivateLink-Verbindung für **QueryGrid**, um VantageCloud Lake mit einer externen Vantage-Umgebung oder einer anderen VantageCloud Lake-Umgebung zu verbinden.


1.  Richten Sie einen externen (Ziel-)Server ein, um Datenkopien zu erstellen.

    Ein Fremdserver kann eine VantageCloud Lake-, VantageCloud Enterprise- oder VantageCore-Umgebung sein.


## 2. Umgebung hinzufügen


Wenn Sie zum ersten Mal eine Datenkopie bereitstellen, wählen Sie eine verfügbare VantageCloud Lake-Umgebung aus dem Menü aus.

Um eine zusätzliche Umgebung hinzuzufügen, gehen Sie zu **Datenkopie** > **Verbindungen** > **Verbindungsplan** und wählen Sie im Menü **+** aus, um eine in QueryGrid eingerichtete Datenquelle zu suchen.

Halten Sie die Informationen zu Ihrem PrivateLink-Endpunkt (AWS oder Azure) bereit, um eine Umgebung zu verbinden.

## 3. Copy Manager installieren


Der Copy Manager ist das auf VantageCloud Lake installierte Softwarepaket, das zum Kopieren von Daten in eine und aus einer Zielumgebung dient.

Wählen Sie eine der verfügbaren VantageCloud Lake-Umgebungen in **Verbindungsplan** aus, in der Copy Manager nicht als installiert angezeigt wird. Wählen Sie im Bereich **Umgebungsdetails** die Option **Copy Manager installieren** aus dem Menü aus.

## 4. Copy Agent installieren


Der Copy Agent ist ein Softwarepaket, das auf den Zielumgebungen VantageCloud Lake, VantageCloud Enterprise oder VantageCore installiert wird, um Daten in die oder aus der VantageCloud Lake-Quellumgebung, auf der der Copy Manager installiert ist, zu kopieren.

Wenn Sie eine Umgebung verbinden, installieren Sie den Copy Agent als Teil der Zielverbindungseinrichtung.

**Hinweis:** Für die Installation der Copy Agent-Software für Daten auf einem VantageCloud Enterprise-System ist Teradata-Support erforderlich.
