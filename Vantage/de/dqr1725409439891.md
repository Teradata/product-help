Nutzen Sie die Option „Bereitstellungen“, um alle Modellversionen eines Projekts anzuzeigen und zu überwachen, die im Status „Produktion“ bereitgestellt sind (*aktive* Bereitstellungen) und entsprechend der Leistung des Modells entsprechende Schritte zu ergreifen. Die Modell-Bereitstellung integriert ein Modell in eine vorhandene Produktionsumgebung, um praktische Geschäftsentscheidungen auf Grundlage von Daten zu treffen.

(Ein stillgelegter Status bedeutet, dass das Modell nicht mehr in der Produktion ausgeführt wird.)

**Bereitstellungstypen**

ModelOps unterstützt die folgenden Bereitstellungstypen:

-   **In-Vantage**

    Die Modellbereitstellung erfolgt innerhalb von Vantage.


-   **Docker-Batch**

    Beginnen Sie eine Batch-Bereitstellung Ihres Modells, indem Sie Eingabedaten aus einer Dataset-Vorlage und Datenverbindung verarbeiten und die Ausgabe in einer Tabelle speichern. Batch-Bereitstellungen können bedarfsgesteuert oder zeitplanbasiert ausgeführt werden.


-   **Docker RESTful**

    Hierbei handelt es sich um die Bereitstellung des Modells als REST-API-Dienst.


Wählen Sie im ModelOps-Dashboard **Bereitstellungen** > eine Bereitstellung aus.
