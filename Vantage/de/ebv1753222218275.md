Als Organisationsadministrator können Sie PrivateLink auf Organisationsebene in der VantageCloud Lake-Konsole aktivieren und den öffentlichen Internetzugriff über URLs deaktivieren.

**Hinweis:** Die Deaktivierung der öffentlichen URL ist für Organisationen mit einem AWS-Konto verfügbar, unabhängig davon, welche VantageCloud Lake-Plattform sie verwenden.

1.  Navigieren Sie zu **Startseite** > **Zugriff verwalten** > **Netzwerksicherheit**.


1.  Geben Sie Ihre zulässigen AWS-Konto-IDs durch Kommas getrennt ein.


1.  Geben Sie die zulässigen Regionen ein und wählen Sie **Speichern** aus.

    **Hinweis:** Teradata-Dienste, die den PrivateLink-Zugriff auf die VantageCloud Lake-Konsole bereitstellen, verwenden die AWS-Region „us-west-2“. Wenn Sie den Zugriff über eine andere Region ermöglichen möchten, verwenden Sie das Menü **Zugriff über Regionen zulassen** und wählen Sie eine Region aus. Aktivieren Sie anschließend beim Erstellen Ihres Endpunkts die Option **Regionsübergreifenden Endpunkt aktivieren** und wählen Sie **USA (Oregon) (us-west-2)** aus dem Menü aus.


1.  Kopieren Sie die von Vantage generierte **Endpunktadresse** und fügen Sie sie im AWS-Portal als Ressourcen-ID oder Alias ein.

    Siehe [Einen von AWS PrivateLink unterstützten Dienst erstellen](https://docs.aws.amazon.com/vpc/latest/privatelink/create-endpoint-service.html)

    **Wichtig!** Deaktivieren Sie **Öffentliche URL** erst, wenn alle Schritte abgeschlossen sind.


1.  Rufen Sie die private URL auf und überprüfen Sie, ob sie korrekt funktioniert.


1.  Während Sie mit der privaten URL bei der Konsole angemeldet sind, gehen Sie zu **Startseite** > **Zugriff verwalten** > **Netzwerksicherheit** und deaktivieren Sie die **Öffentliche URL**.

    Das Deaktivieren der öffentlichen URL auf Konsolenebene hat keine Auswirkungen auf die Netzwerkeinstellungen der Umgebung.
