Cree conexiones de conjunto de datos para asegurarse de que los proyectos, las plantillas de conjuntos de datos y las versiones de modelos que entrena utilicen datos de una base de datos específica. Puede crear varias *conexiones de conjunto de datos personal*, incluida una predeterminada, para asegurarse de que sus conjuntos de datos y modelos utilicen la misma base de datos. Utilice las conexiones de conjunto de datos personal predeterminadas al crear plantillas de conjuntos de datos y entrenar modelos; no son obligatorias, pero se recomienda su uso con proyectos. Las conexiones de conjunto de datos personal predeterminadas son necesarias para importar versiones de modelos.

Los usuarios administradores de ModelOps pueden crear *conexiones de conjunto de datos de servicio* para su uso al desplegar modelos. Se recomienda crearlas antes de desplegar un modelo, ya que se solicitará que se especifique una conexión de conjunto de datos de servicio.

El administrador de ModelOps proporciona las credenciales necesarias para crear una conexión de conjunto de datos.
