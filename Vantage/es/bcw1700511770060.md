Un administrador de la organización puede aprovisionar el servicio de Copia de datos en VantageCloud Lake con la ayuda del soporte de Teradata.

Cree una solicitud de soporte en [https://support.teradata.com](https://support.teradata.com) para obtener ayuda con los siguientes requisitos:

-   Configure **QueryGrid** para conectar la Copia de datos entre entornos.


-   Instale el gestor de copias en VantageCloud Lake.


-   Cree un [endpoint de interfaz](dvp1707442265467.md) de AWS o Azure entre entornos.


-   Instale el agente de copia en el entorno de destino. Puede ser un entorno de VantageCloud Lake, VantageCloud Enterprise o VantageCore.

    **QueryGrid** es el método de copia admitido.


-   Conecte entornos.


**Notas**:

-   No se puede conectar el gestor de copias a un sistema que ya esté conectado a un **gestor de Copia de datos** diferente.


-   No se puede conectar el gestor de copias a más de un sistema.


Consulte [Habilitar servicio Copia de datos](https://docs.teradata.com/access/sources/dita/topic?dita:topicPath=zmv1694773546514.dita&utm_source=console&utm_medium=iph) para obtener instrucciones detalladas sobre el aprovisionamiento para plataformas no compatibles con Lake.

## 1. Requisitos previos de QueryGrid


Es necesario realizar la siguiente configuración **QueryGrid antes de poder habilitar Copia de datos:**

1.  Aprovisione **QueryGriden su entorno de VantageCloud Lake.**

    Desde el entorno, vaya a **Gestionar datos** > **QueryGrid**. Consulte [Despliegue de QueryGrid](wrw1640282047881.md).


1.  Cree una conexión de enlace privado para **QueryGrid** para conectar VantageCloud Lake a un entorno de Vantage de destino o a otro entorno de VantageCloud Lake.


1.  Establezca un servidor externo (de destino) para realizar copias de datos.

    Un servidor externo puede ser un entorno de VantageCloud Lake, VantageCloud Enterprise o VantageCore


## 2. Agregar un entorno


Para el primer aprovisionamiento de Copia de datos, seleccione un entorno de VantageCloud Lake disponible en el menú.

Para agregar un entorno adicional, vaya a **Copia de datos** > **Conexiones** > **Mapa de conexión** y seleccione **+** para buscar una fuente de datos establecida en QueryGrid desde el menú.

Debe tener la información del endpoint de enlace privado (AWS o Azure) para conectarse a un entorno.

## 3. Instalar el gestor de copias


El gestor de copias es el paquete de software instalado en VantageCloud Lake para copiar datos hacia y desde un entorno de destino.

Seleccione uno de los entornos de VantageCloud Lake disponibles en **Mapa de conexión** que no muestre el gestor de copias como instalado. En el panel **Detalles de entornos**, seleccione **Instalar gestor de copias** en el menú.

## 4. Instalar el agente de copia


El agente de copia es el paquete de software instalado en los entornos de VantageCloud Lake, VantageCloud Enterprise o VantageCore de destino para copiar datos hacia y desde el entorno de VantageCloud Lake de origen que tiene instalado el administrador de copias.

Cuando se conecta a un entorno, el agente de copia se instala como parte de la configuración de la conexión de destino.

**Nota:Se requiere soporte de Teradata para instalar el software del agente de Copia de datos en un sistema VantageCloud Enterprise.**
