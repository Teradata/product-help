Utilice **Copia de datos** para copiar datos entre dos entornos VantageCloud Lake o entre VantageCloud Lake y un entorno VantageCloud Enterprise o VantageCore. Copia de datos utiliza **QueryGrid** como método de copia para realizar copias completas o parciales de tablas.

Copia de datos admite las siguientes conexiones *bidireccionales* de bases de datos de origen y destino:

|Origen (local)|Destino (externo)|
|---------------|------------------|
|VantageCloud Lake on AWS|VantageCloud Lake on AWS|
|VantageCloud Enterprise on AWS|VantageCloud Lake on AWS|
|VantageCore (local)|VantageCloud Lake on AWS|
|VantageCloud Lake on Azure|VantageCloud Lake on Azure|
|VantageCloud Enterprise on Azure|VantageCloud Lake on Azure|
|VantageCore (local)|VantageCloud Lake on Azure|

**Importante:** Copia de datos requiere que **QueryGrid** esté aprovisionado y configurado para conectarse a los sistemas Vantage de destino: VantageCloud Lake, VantageCloud Enterprise o VantageCore.
