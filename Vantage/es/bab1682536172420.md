Desencadenadores admitidos en tablas del sistema de archivos de objetos
-----------------------------------------------------------------------

VantageCloud Lake ahora admite desencadenadores en tablas en el sistema de archivos de objetos.

Aumento y reducción del clúster principal
-----------------------------------------

Cuando se aprovisiona inicialmente un entorno, se especifica el tamaño de la instancia del clúster principal.

Después de aprovisionar un entorno, se puede aumentar o reducir el tamaño de la instancia original hasta en dos tamaños de instancia.

Aumente para incrementar a corto plazo la carga de trabajo y las tareas con uso intensivo de datos para mantener un rendimiento óptimo o acelerar el procesamiento.

Reduzca durante los períodos de demanda inferior para evitar pagar por recursos no utilizados.

Para aumentar o reducir el clúster principal, debe enviarse una solicitud.

Expansión de almacenamiento de bloques para el clúster principal
----------------------------------------------------------------

Cuando se aprovisiona inicialmente un entorno, se especifica la cantidad de almacenamiento de bloques que se quiere para el clúster principal.

Después de aprovisionar un entorno, se puede agregar almacenamiento de bloques al clúster principal en incrementos de 1 TB.

La expansión de almacenamiento de bloques para el clúster principal no se puede reducir (revertir) después de incrementarse.

Para agregar almacenamiento de bloques al clúster principal, debe enviarse una solicitud.

Notificaciones por correo electrónico de alerta de consumo
----------------------------------------------------------

Establezca alertas para notificar, a usted y a otras personas, que el consumo se acerca a un umbral. Se puede establecer más de una alerta para cada organización.

Índices de unión de tabla única en tablas del sistema de archivos de objetos
----------------------------------------------------------------------------

A partir de esta versión, los índices de unión de tabla única (STJI) se admiten en las tablas del sistema de archivos de objetos en el índice principal.

Soporte de plataforma de almacenamiento de objetos externa
----------------------------------------------------------

Ahora se admiten las siguientes plataformas de almacenamiento de objetos externas:

-   Cloudera

-   Cloudian HyperStore

-   IBM Cloud Object Store (IBM COS)

-   RedHat Ceph

-   Scality Ring

-   VAST Data

-   Vcinity VAccess

Valor de registro DBQL predeterminado
-------------------------------------

Para el registro DBQL, el registro detallado es el valor predeterminado en VantageCloud Lake. Para usuarios o cuentas que ejecutan cargas de trabajo tácticas, la recomendación es habilitar el registro de resumen/umbral DBQL, de modo que el volumen de registro sea nominal. Se puede acceder al registro de resumen mediante td\_metric\_svc.dbqlsummaryV.
